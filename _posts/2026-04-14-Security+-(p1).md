Security controls is a topic that covers the different aspect of controlling risks and hazards of cyber security.

These are used to prevent a security event from occurring, minimize the impact of an event or limit/mitigate the damage that occurs if said event takes place.

There are different security controls that are split into a few broad categories.

- **Technical controls**
- **Managerial controls**
- **Operational controls**
- **Physical controls**
  
**Technical Controls** — These are controls that are set up within the system that you are protecting, such as firewalls, antivirus or other bespoke methods. These are software based.

**Managerial Controls** - These controls are administrative, “paper” based such as legislation, operating procedures, training courses, awareness programs.

**Operational Control**s — These are controls that are implemented by "People” not systems, these include Security guards, Calling the authorities in the event on an incident, reporting issues, Patrolling, incident hand over briefing etc.

**Physical Controls** — These controls are in the physical world and block/deter physical access to areas, such as locks, barriers key cards etc

**Further control descriptors:**

There are also **Deterrent, detective, Corrective, Compensating & Directive**. Additionally remember that many cover multiple areas (e.g. CCTV could be a Deterrent & detective)

**Deterrent** — Something that deters action rather than blocks it, Examples would be signage, CCTV, Lighting, or disciplinary action

**Detective** — Allows you to be able to garner information on security events, CCTV, digital logs, log in screens, intruder alarms. It may/may not prevent access to the site/system.

**Corrective** — Once an event has taken place what can be done to mitigate the issue, think Fire extinguishers, emergency procedures, backup /restore data.

**Compensating** — these are additional resources/means in which you can control & mitigate security events. Think about having in a backup generator if the power cuts out. Their use could be temporary. This could also be that you disable some functionality of application if there are security concerns whilst you wait to get it patched. It could also mean hiring additional staff to prevent exploitation of system weaknesses.

**Directive** — This is where a “subject” is directed towards security compliance, “Do This” etc. This seems very similar to both Managerial/Operative controls., After some research Directive Control is more specifically explicit and definitive. Rather than Managerial is more broad and includes multiple facets rather than just dealing with directing people.
